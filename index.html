Tic Tac Toe
   

The code for this exercise is at:

https://github.com/marceliotstein/tictactoe.

This was an easy coding task but a difficult design/layout task. The toughest challenge was to create a game board and control panel that makes good use of all available space on a wide variety of screen sizes. I used a combination of absolute and relative positioning, as well as a combination of viewport-based and media-query-based responsive technique. The result may skew at abnormally wide or narrow screens, but works correctly on the various device sizes I was able to test with.

Another interesting decision I had to make was whether to implement the game board data structure with a single or multi-dimensional array. If there were any chance that this code would have to be adapted to a 4x4 or greater grid, I would have chosen a 2-dimensional array for extensibility. However, the 3x3 grid is a basic principle of Tic Tac Toe, so I used a 1-dimensional array for maximum simplicity and efficiency.
